# Repository guide for coding agents

- **Rust**：`crates/feishu-approval-tool`（库 `lib.rs` 暴露 **`api_paths`**：`src/api_paths.rs`，统一 `/open-apis/approval/v4/...` 常量与路径拼接；CLI `main.rs`；`cli/handlers/`：`mod.rs`（含 `resolve_instance_create_form`、`exec_normalized`/`dispatch_api`）、`approval.rs`（`dump --data-only`）、`instance.rs`、`task.rs`、`comment.rs`、`util.rs`、`wizard.rs`（`instance create --wizard`）；`cli/json_util.rs`（含 **`validate_instance_create_extra_patch`**，防 `node_approver_*` 内层误用 `node_id` 等））。
- **AI 文档**：根目录 **`docs/AI.md`** 为面向模型的**唯一**操作手册（**AI-first**：CLI 与手册优先服务 Agent 调用），**§1「阅读顺序」** 为「文档怎么读」唯一详表；改 CLI 行为时同步更新 **`docs/AI.md`**、**`SKILL.md`**（frontmatter `description`、全局参数表若有变），并核对 **`README.md`** 与 **`AGENTS.md`** 指向是否仍对。**`util`**（离线为主）：`form-string`、`validate-widgets`、`extract-widgets`、`scaffold-widgets`、`init`、`doctor`。**`instance create`**：`--validate-against-json`、`--wizard`、`--template`。**`task reject`**：`--task-id` / `--task-ids` / `--batch-json-file`。**`task search`**：`--json-file` 为对象 + 可选 `--pending-only` / `--task-status` / `--search-user-id` 浅合并。完整以 **`docs/AI.md`** 为准。**`OpenApiClient::request_json`**：HTTP 2xx 且 JSON 顶层 **`code` ≠ 0** 时返回 `Err`，stderr 可含 **`docs/AI.md` §7** 类提示。
- **勿恢复**已删除的 `recipes/`、仓库级 **`approval-code-map.md`**、根目录 **`approval-code-map.local.md.example`**（模板现为 **`docs/approval-code-map.local.template.md`**；**`util init`** 用 `include_str!` 引用它）。映射约定见 **`docs/AI.md` §8**（无 **`approval-code-map.local.md`** 且用户只给审批名时：**必须**索要 **`approval_code`**，用户提供后 **`util init` + 编辑表格** 落盘）。示例 JSON 在 **`docs/examples/`**。**`approval-code-map.local.md`** 不入库。
- **验证**：`cargo test -p feishu-approval-tool`、`cargo clippy -p feishu-approval-tool --all-targets`。
